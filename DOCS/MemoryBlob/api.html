Module memoryblob

Auxiliary library for creating memory blobs, in order say to recycle or share resources, especially between native plugins.

To use the plugin, add the following in build.settings:

plugins = {
   ["plugin.MemoryBlob"] = { publisherId = "com.xibalbastudios" }
 }

Sample code is available (TODO). (Probably won't be much... this is a pretty boring library on its own)

Note that the MemoryBlob methods below apply to any blob, not only those created by this plugin. In particular, other plugins may create their own blobs via the BlobXS API: see here, in particular ByteUtils.h and Blob.cpp.

Functions

GetVersion ()
Gets the blob version that will be associated with blobs created by New. Blobs "in the wild" might have been created with an older blob API; a program can handle such mixtures by comparing blobs' versions against this value.

This plugin is intended to track the latest version.

Returns:

    uint Version, ≥ 1.
IsBlob (object[, type])
Indicates whether an object (which need not originate from this plugin) is a MemoryBlob.

Parameters:

  • object Object to check.
  • type string If present, refine the query to only consider blobs of a given type, cf. New.

Returns:

    boolean Is object a blob?
New (opts)
Creates a new memory blob.

Parameters:

  • opts ptable, uint or nil Blob creation options.

    When opts is a table, it may contain the following options:

    • alignment: If specified, the memory alignment, which must be a multiple of 2, ≥ 4. The blob's memory will start at an address that is a multiple of this value, which is useful and / or needed e.g. for SIMD operations. By default, blobs use the Lua allocator's alignment.
    • resizable: If true, the blob can be resized (via the native API). Off by default.
    • size: Blob size, in bytes. In the case of resizable blobs, this is the blob's initial size (≥ 0; if absent, 0); otherwise, this is mandatory and specifies the fixed size > 0.
    • type: A string that will be used as the blob userdata's metatable name; if absent, a default is used. Blobs themselves make no further use of this value; rather it is exposed as a convenience, e.g. so plugin authors can identify their own blobs via IsBlob.

    If opts is an integer (> 0), it specifies the fixed size of the blob to create.

    For any other value of opts, a resizable blob will be created.

Returns:

    MemoryBlob The new blob.
MemoryBlob:Clone ()
Creates a clone of the blob.

The clone will have the same alignment, resizability, size, and type as the original blob, as well as a copy of the original's bytes.

The clone begins unlocked.

Returns:

    MemoryBlob Cloned blob.
MemoryBlob:GetProperties ([out])

Populates a table with the following blob properties:

  • alignment: As per New, an integer ≥ 0, where 0 indicates default alignment.
  • resizable: As per New, a boolean indicating resizability.
  • version: The API version used to create this blob, cf. GetVersion.

Parameters:

  • out table If provided, this table is populated; otherwise, a new one is created.

Returns:

    table Properties table.
MemoryBlob:IsLocked ()
Indicates that the blob is locked for writing (via the native API).

If this is true, the blob is not meant for general use and will be rejected by well-behaved external code (some utilities on the native side provide assistance here).

Returns:

    boolean The blob is locked?
