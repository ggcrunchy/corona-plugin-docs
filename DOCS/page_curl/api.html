Module page_curl

This module provides a page curl widget.

To use the plugin, add the following in build.settings:

   ["plugin.page_curl"] = { publisherId = "com.xibalbastudios" }

PageCurlWidget:Begin ()
Puts the widget in batching mode.

The motivation is to avoid incurring the (possibly expensive) overhead that comes with calling methods that update the widget's state, e.g. creating new shaders, deferring it until the batch is committed, cf. PageCurlWidget:Commit.

When batching is already in progress, this is a no-op.

N.B. #1 This is unnecessary when setting the touch direction (since this affects no graphics state) or calling read-only methods such as PageCurlWidget:GetGrabRegion.

N.B. #2 Until the batch has been committed, the widget will be in an irregular state. During this period, polling the widget's curl properties or calling getter methods will yield incorrect results. Touch controls will likewise be out of sync.

See also:

    PageCurlWidget:SetDirection
PageCurlWidget:Capture (target)
Captures a target and sets it as the current page.

This will commit any batch in progress, cf. PageCurlWidget:Begin.

Parameters:

  • target DisplayObject, ContentBounds or nil If target is a display object, this has display.capture semantics.

    If target is nil instead, display.captureScreen semantics are chosen instead.

    Otherwise, display.captureBounds semantics are used and target is assumed to be a table compatible with object.contentBounds.

PageCurlWidget:DetachSnapshot ([into])
If the current page is a snapshot, removes it from the curl widget's control. Aside from changing parents, it keeps its most recent state, but the curl widget is no longer able to affect it, and vice versa.

If the current page is not a snapshot, this is a no-op. Otherwise, the curl widget is cleared; this will commit any batch in progress, cf. PageCurlWidget:Begin.

Parameters:

  • into DisplayGroup Group that will receive the snapshot. If absent, the stage is used.

See also:

  • PageCurlWidget:PrepareSnapshot
  • PageCurlWidget:SetBlankRect
PageCurlWidget:EnableEdgeTinting (enable)
Enables or disables edge tinting.

After the page has been curled a bit, its back side will be visible. Edge tinting applies some effect on or near the edges of this back leaf.

Edge tinting is enabled by default.

Parameters:

  • enable bool Enable tinting?

See also:

    PageCurlWidget:SetTintMethod
PageCurlWidget:EnableExpand (enable)
Enables or disables expansion.

When the page is curled in most directions, it overflows the boundaries of the underlying display object. To accommodate this, the object can be expanded and its textures are read with this in mind.

When disabled, out-of-bounds pixels are clipped. This might be useful, for instance, if the widget is partially obscured.

Expansion is enabled by default.

Parameters:

  • enable bool Enable expansion?
PageCurlWidget:EnableInnerShadows (enable)
Enable or disable inner shadows.

Once the page begins curling, a light shadow is applied to the curled-up part of the front side of the page.

Inner shadows are enabled by default.

Parameters:

  • enable bool Enable inner shadows?

PageCurlWidget:SetDirection (dir)
Updates the curl direction followed by touch, enabling touch in the appropriate region.

Parameters:

  • dir string or nil If absent, touch is disabled.

    Otherwise, this may be any of "bottom_to_top", "left_to_right", "right_to_left", or "top_to_bottom". (If dir is anything else, this is a no-op.) Appropriate touch behavior will be activated, and the grab region will change to reflect this choice.

NewPageCurlWidget ([opts])
Constructs a new page curl widget.

The PageCurlWidget type derives from GroupObject. Well-behaved code should leave its children alone, but may otherwise treat is a normal display group.

Page curl widgets have edge_x, edge_y, angle, and angle_radians properties, in addition to those common to any display group. These may be both read and written, say as local angle = my_curl_widget.angle or my_curl_widget.edge_x = .3.

The edge_x and edge_y properties mark a point along the edge where the page begins to curl, the so-called "curling axis". (Any such point is fine.) The coordinate system goes from left-to-right horizontally and top-to-bottom vertically, with a value of 0 corresponding to left or top, and 1 to right or bottom.

The angle property describes the direction of the outward-facing normal to this edge, i.e. along the direction of curl. Angles are denominated in degrees, going clockwise. The angle_radians property, which instead uses radians as its unit, is provided as a convenience.

The widget is designed to be used both programmatically and through touch.

When touch is active, the widget will dispatch the "page_dragged" event during any movement that updates the page. The curl widget can be accessed from a listener as event.target.

Parameters:

  • opts table

    Constructor options. Fields:

    • left: When present, this is the coordinate of the x-coordinate of the widget's left-hand side... (The default is 0.)
    • top: ...and this is the y-coordinate of its top. (The default is 0.)
    • width: The width of the initial (blank) rect... (When absent, a default is used.)
    • h: ...and the height. (Ditto.)
    • size: A positive integer used to size the grabbable area, for touch purposes. This describes how thick the grab region is along the curl direction. In addition, it says how much total cushion to provide along the relevant edge of the page, in order to include touches that while out-of-bounds are still "close enough".

Returns:

    PageCurlWidget New page curl widget.
